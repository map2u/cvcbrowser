1.0 Overview of Credit Valley Watershed and Well-Being Browser 1.0

1.1 Purpose

This project aims to show how improving the health of the natural environment also improves the well-being of local residents of and visitors to the watershed. With input from the local residents we will develop a set of indicators and a map tool to effectively communicate these relationships. This map-based tool will be made available to the public to explore. Ultimately, it will be used in conducting scenario analyses to see how different management decisions affect the well-being of residents.This project addresses requirements of data sharing and monitoring specified in Annex 4 of the Canada-Ontario Agreement (COA). It provides a collaborative mechanism among researchers at York University, and the Credit Valley Conservation Authority (CVC) to monitor environmental management decisions in relation to the human well-being.

1.2 Partners

Development and maintenance of the Credit Valley Watershed and Well-Being system is led by Professor Martin Bunch out of the SpEAR lab at the Faculty of Environmental Studies at York University. This phase of the project is sponsered by Credit Valley Conservation Authority, Water, Economics, Policy and Governance Network, and Canadian Water Network.

1.3 Software and System Design

Credit Valley Watershed and Well-Being Browser is a web-distributed geographic information system (web-GIS). It is composed of: (i) Free and Open Source System (FOSS) code written for the Credit Valley Watershed and Well-Being project, (ii) FOSS software modules developed by others and linked together to form the overall Credit Valley Watershed and Well-Being Browser system and (iii) a Googlemaps mashup. Credit Valley Watershed and Well-Being Browser was developed using FOSS development tools. (The only component that may potentially be propriety is the spatial data required to delineate watershed and subwatershed boundaries).

2.0 The Credit Valley Watershed and Well-being Browser Interface

The CVC Watershed and Well-being Browser interface consists of four main sections (Figure 1 below): 1) the Header Bar, 2) the Login tab, 3) Visual header tabs, and 4) the Map Viewer tab. A brief description of these components is provided below, and instructions on how to use the various functions accessible via these interface components can be found in Section 3 (Accessing Credit Valley Watershed and Well-being Browser), Section 4 (Using Primary CVC Watershed and Well-being Browser Interface Components), and Section 5 (Credit Valley Watershed and Well-being Browser Core Functions).

Figure 1: Four main sections of the CVC Watershed and Well-being Browser interface: 1) the Header Bar, 2) the Login Tab, 3) Information Tabs, and 4) the Map Viewer Tab.

2.1 Header Bar

The Header Bar runs across the top of the Credit Valley Watershed and Well-Being Browser interface (#1 in Figure 1). Above the Header Bar are the logos of the principle project partners. Clicking on these will open a browser window that loads the home page of these partners. Below the Sponsors is the Project Banner displaying the project name, clicking this banner at anytime will return the user to the home page. The Header Bar extends across the Credit Valley Watershed and Well-Being interface.

The left side of this strip contains drop down menus for Well-Being, Ecosystem Services, and Your Stories tabs. Selecting one of these tabs will bring the user to a map viewer that contains relevant open source information about the subject ranging from written, visual, and geographic information.

2.2 Login Tab

Located on the right side of the header bar, the user will find the "Login" and "about" tab. This is where the user will register for or log into their account, retrieving a lost password, or find information about the project itself.

Overall, the Header Bar is a stable component, but when a user is logged in the information at the right of the bar changes to display the username of the currently logged-in user, a logout button and a function to change the user's password.

2.3 Information Tabs

These tabs are a visual representation of the tabs within the header bar. Clicking one of these tabs will bring the user to written information about the particular topic and where the user is able to find many more resources related to the topic of choice.

My Stories:

    Selecting the "My Stories" tab will bring the user to an informative page explaining to the user how to upload a brief story about an experience with a particular ecosystem that they want to share publicly. This feature allows a registered user to write a story ranging from 100-200 words and the option to include photos depicting the ecosystem. Guidelines to writting a story can be found Here.

2.4 Map Viewer

The Map Viewer (#4 in Figure 1) is located immediately below the header bar in the middle of the interface, between the Watersheds Info panel and the Search Bar. This is a googlemaps mashup that combines spatial data from maps.google.com (Google Streets, Hybrid, Satellite and Terrain, etc.) with data provided by the Cedit Valley Watershed and Well-Being system (e.g., watershed polygons, subwatershed polygons) and data entered by users (Addresses, overlays in the database). The Map Viewer provides basic geographic information system (GIS) capability to pan, zoom in, zoom out, select an entity, add data, turn on (view) or turn off various data layers, and access data management. On the left side of the map viewer there are functions to draw lines and shapes and to edit layers. In the strip ranging across the top of the map viewer, you will find buttons to reset the Map Viewer window and previous and next map options. On the right hand side of this strip, there is a dropdown menu in which you can select Map Layers where you will find the OverLays you wish to make active. (Some of this functionality operates in conjunction with other interface elements of the Credit Valley Watershed and Well-Being Bowser system). See "Using the Map Viewer" below, for more information.

3.0 Accessing Credit Valley Watershed and Well-Being Browser

3.1 Types of accounts

Many functions of the Credit Valley Watershed and Well-Being system can be accessed by the general public without needing a user account to log in to the system. Only My Story upload, data upload, data editing, and data deleting functions and account management require a user account.

There are two types of accounts in the Watershed and Well-Being Web-GIS system: Volunteer accounts and Administrator accounts. Volunteer accounts are intended for users of the system who participate in providing information about the benefits of local watershed and human health well-being by contributing to open source data and stories about their own experiences. Administrator accounts are intended for personal that need to administer the data generated, assign new accounts, remove and add data, etc.

3.2 Logging in to the system

To log into the Credit Valley Watershed and Well-Being Browser system a user must have a valid user name and password. Valid users can log into the Credit Valley Watershed and Well-Being system by clicking on the "Login" button in the upper right corner of the Watershed and Well-Being interface. In the dialog box that appears, users must enter their username into the "Login Name" field and their password into the "Password" field, then click the "OK" button. Please note that the system will take a couple of seconds to refresh itself after a user logs in.

If you have forgotten your password, you can request a password by clicking the "Forgot Password" button, and entering your Login Name and email address in the dialog box that appears.

3.3 Logging out of the system

Users can log out of the Credit Valley Watershed and Well-Being Browser system by clicking on the "Logout" button at the upper right of the Watershed and Well-Being interface. Please note that the system will take a couple of seconds to refresh itself after a user logs out.

4.0 Using Primary Credit Valley Watershed and Well-Being Interface Components

4.1 Using Location Search

To find a location in the Credit Valley Watershed and Well-Being system: On the strip located on the left side of the Map Viewer, type an address or other location information in the general area of your monitoring station in the "Address / Postal Code" search box. Then click the "Search" button.

Note that you can type in postal codes, street addresses, geographic coordinates, street intersections, etc. Here are some examples that will find York University in Toronto, Ontario, Canada:

  1. M3J 1P3
  2. 4700 Keele Street, Toronto, ON
  3. Keele and Finch, Toronto, ON
  4. 43.76N, 79.51W

Once you submit your address, the system will pan and zoom to that location and place a temporary blue icon there. Allowing the user to then use map layers in Figure 2.2 to select different overlays to find information on their watershed and well-being.

4.2 Using the Map Viewer

Panning (moving around the Map Viewer display):

Zooming in or out (changing the scale): Zoom in or out by clicking on the + (zoom in) or - (zoom out) button in the upper right corner of the Map Viewer window (#6 in Figure 2). Double-clicking will centre and zoom the Map Viewer window on the selected feature (#7 in Figure 2). #1 in Figure 2.1 shows the selected watershed name and #2 in Figure 2.1 shows the subwatersheds belonging to the selected watershed.

  • The Overlay List is located on the right side of the Credit Valley Watershed and Well-Being interface, below the Header Bar in the Watersheds Map layer Panel under "Map Layers".
  • Under the overlays list, the user is presented with an expandable/collapsible list of map layers within the Credit Valley Watershed.

5.0 Credit Valley Watershed and Well-Being Core Functions

The Credit Valley Watershed and Well-Being system is intended to be an open source information system for the purpose of sharing information related to watershed connectivity and human well-being. To support this the Credit Valley Watershed and Well-Being system allows a user to:

  • create a record for watershed connectivity composed of (a) a point location represented by an icon in the map viewer and (b) data records in the database,
  • upload data to the records associated with their locations(s),
  • edit/modify layer data that has previously been entered ,
  • Download data that includes layer information and perosnal analysis of human health and well-being.

5.1 Creating a map Layer

How to create and upload a new Layer:

Create a new layer when:

  • There is relevant information that can contribute to the knowledge and benefit of the general public.
  • There is a unique and/or major feature and/or condition.

To create a map layer:

  • Log into the Credit Valley Watershed and Well-Being system.
  • Select “Add Spatial File” on the panel options on the right side of the map.

To upload data:

  • Once selected, the user can upload the new layer by clicking on the "browse" button and uploading the required shapefiles (.shp, .shx, .dbf, .prj) #1(Figure 3)
  • Once the shapefiles have been uploaded, the projection must be selected for proper spatial representation.
  • Once the user is satisfied that they have completed their data entry task they will click on the "upload" button in the bottom-left of the window #2(Figure 3). Alternatively, they may use the "exit" or "Cancel" button to either set the window to its original condition or exit from the data upload window.

Figure 3: Creating and uploading a map layer.

5.2 Editing Data

How to edit a new Layer:

  • When uploading data, the user may want to set visual characteristics to the map in order to display information.
  • Select the "Upload File List" located on the option panel on the right side of the map interface.
  • Once selected the user must click on their uploaded shapefile and select the option "Set Upload File Settings" #1.(Figure 3.1)
  • With the "set Upload File Settings" selected, the user can input the thematic features and the boundary demension for defining the dataset layer. The user must first select the correct data source and data field name before making changes to the data set #2.(Figure 3.1)
  • Once the user is satisfied with the results, select "Apply" to submit the changes to the dataset #3.(Figure 3.1) CAUTION: submitting edited data will overwrite the original dataset

Figure 3.1: Changing the thematic display characteristics.

5.3 Geometric Features

How to add geometric features:

Users can access geometric features through the "Your Story" tab option displayed in (Figure 3.2)

Figure 3.2: Displaying geometric features in the Your Stories window.

  • Users can add geometric features (lines, polygons, squares, circles, and markers) to a map layer in order to outline particular characteristics of a map.
  • located on the left side of the "your story" map interface, users can select the geometric features by clicking on the edit map tool icons the represents the desired shape as shown in Figure 3.2)
  • Once the feature is selected, the user can manually draw or place the feature in the desired location, where the "Share Your Storey" panel will appear, as shown in (Figure 3.3)
  • Within this panel, users will be able to provide information concerning a feature name and summary. The user can also include an image, video, and a pdf or html story file within the defined feature for further discription and information.

How to add a visual marker to a map layer:

  • Users can add a visual marker feature to highlight a specific area or interesting feature on a map.
  • located on the left side of the map interface below the geometric features in Figure 3.2, users can select the "draw a marker" icon by clicking on it and placing the marker in the desired location on the map.
  • Once the marker has been placed, the "Share your Story" panel will appear (as shown in Figure 3.3). The user can then add the Story information pertaining to that marker (Name, Summary, Images, etc).
  • Clicking "save" will add the marker to the map layer.

Figure 3.3: Adding geometry to a story within a map layer.
